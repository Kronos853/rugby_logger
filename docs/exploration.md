# Exploration: Sports Team Statistics & Video Tagging

> Обновлено: 2025-06-22  
> Статус: **готово к черновику ТЗ** (основные решения приняты)  
> Стек — **зафиксирован** (см. `openspec/changes/sports-video-logger-mvp/design.md`)

## Цель проекта

Программное обеспечение для:

1. **Ввода статистики** спортивной команды (просмотр видео + формирование таблицы событий)
2. **Хранения данных** в локальной базе: игроки, команды, матчи, события
3. **Анализа** статистики — встроенными средствами и с помощью ИИ (фаза 2)

### Продукт в целом

| Аспект | Решение |
|--------|---------|
| Пользователи | **Однопользовательское** приложение |
| Команды | **Много команд** в одной базе |
| Хранение | **Локально** на ПК |
| Экспорт | **CSV** (бэкап, ручная загрузка во внешний ИИ) |
| Платформа | **Браузер** (два монитора через браузер — достаточно) |
| Ввод данных | **Post-match**, по **локальным видеофайлам** |

### Универсальность и таксономия

- Программа **универсальная** — у каждого вида спорта свой набор категорий, действий, комментариев
- Есть **шаблоны спорта** (Sport Template); первый — **регби**
- **Категория → много Действий** (1:N)
- У действия опциональный **результат**: успех / провал
- У действия — **комментарии**: **кнопки-подсказки** (быстрое заполнение) + **ручной ввод** для особых случаев
- Таймы/периоды — **вручную**, структура зависит от вида спорта (в шаблоне)

### Модель данных

| Сущность | Решение |
|----------|---------|
| **Команда** | **Справочник команд** — названия вносятся заранее; перед разметкой матча **выбираются команды** (своя + соперник) |
| **Игрок** | Привязан к команде; имя; позиция — опционально |
| **Команда без игроков** | Запись в справочнике **только с названием** — для командных событий и когда неясно, к кому отнести |
| **Состав (Squad)** | Именованный набор игроков+позиций на турнир/сезон; у каждого игрока флаг **основной / замена** (`LineupRole`) — **только для удобства разметки**, не для отчётов в MVP |
| **Состав матча (MatchLineup)** | Снимок Squad на матч; флаг основной/замена копируется и **можно править** под конкретный матч |
| **Матч** | Дата, турнир, счёт, таймы/периоды — вручную; **две команды из справочника** |
| **Событие** | Независимое; без координат; без связей; субъект — **игрок** или **команда** (из справочника) |
| **Видео** | Несколько файлов на матч возможно; **не сохраняем** — только для наполнения статистики |
| **Таймкод** | **Секунды** (долей секунды не нужно) |

### Экспорт CSV

Колонки (фиксированный формат):

| Колонка | Описание |
|---------|----------|
| `Тайм` | Период / тайм (вручную) |
| `Время` | Таймкод в видео (секунды) |
| `Игрок/Команда` | Имя игрока или название команды из справочника |
| `Категория` | Категория действия |
| `Действие` | Тип действия |
| `Результат` | Успех / провал (может быть пустым) |
| `Комментарий` | Текст (из подсказки и/или ручной ввод) |

### Видео и разметка

- Видео идёт → **«Новое событие»** → фиксируется текущее время (секунды)
- Автонарезка клипов — **не нужна**
- **Перемотка по клику** на строку события — **можно добавить** (желательно)
- Два монитора — **основной режим**; один экран — **опциональный fallback** (PiP / встроенный плеер)

### Режимы UI

1. **Справочники** — команды, игроки, шаблоны
2. **Заполнение статистики** — разметка матча по видео
3. **Отчёты** — фаза 2
4. **Общение с ИИ** — фаза 2

### Аналитика и ИИ

| Функция | Когда |
|---------|-------|
| Ввод статистики + CSV | **Фаза 1 (MVP)** |
| Сводки по игроку (матч / сезон), успех/провал | Фаза 2 |
| Конструктор отчётов (гибкая сборка отчётов из метрик) | Фаза 2 |
| ИИ-чат через облачный API (лёгкий/средний уровень) | Фаза 2 |
| Авто-детекция на видео (CV) | Не в scope |

ИИ: **облачный API**. Ручной путь: CSV → внешний ИИ.

```
┌─────────────┐     ┌──────────────┐     ┌─────────────┐
│SportTemplate│────►│  Category    │────►│   Action    │
│  (Регби-7)  │ 1:N │              │ 1:N │             │
└─────────────┘     └──────────────┘     └──────┬──────┘
                                                │ 1:N
                                         ┌──────▼──────┐
                                         │CommentTemplate│
                                         └─────────────┘

┌─────────┐ 1:N ┌────────┐       ┌───────────────┐
│  Team   │────►│ Player │       │ MatchLineup   │
│(справоч-│     │(опц.)  │       │ (состав)      │
│  ник)   │     └────────┘       └───────┬───────┘
└────┬────┘                              │
     │  команда может быть без игроков    │
     │ 1:N                                │
     └──────────► Match ◄─────────────────┘
                  │ team_home, team_away (из справочника)
                  │ date, tournament, score
                  │ periods (вручную, из шаблона)
                  │
                  │  видео: временно при разметке,
                  │  несколько файлов, не в БД
                  │
                  └──► Event
                         • player ИЛИ team (из справочника)
                         • action, outcome (опц.)
                         • period / тайм
                         • timestamp (секунды)
                         │
                         └──► comment (текст)
```

### Справочник команд

Единый справочник **Команды**:
- вносятся **названия** команд заранее
- команда **может иметь игроков** (основная) или **только название** (для командных/неясных событий)
- перед разметкой матча **выбираются команды** из справочника (своя + соперник)
- при вводе события субъект — **игрок из состава** или **команда целиком** (запись без игроков)

```
Справочник «Команды»
┌──────────────────┬─────────────┐
│ Барсы U-15       │ есть игроки │
│ Спартак U-15     │ есть игроки │
│ Команда (общ.)   │ без игроков │  ← командные действия
└──────────────────┴─────────────┘
         │
         ▼
   Матч: выбор двух команд → состав → разметка
```


## Текущий прототип (референс UX)

| Файл | Назначение |
|------|------------|
| `video_screen.html` | Полноэкранный видеоплеер на втором мониторе |
| `control_panel.html` | Пульт разметки на первом мониторе (основной референс) |
| `analyse.html` | Одноэкранный вариант с Picture-in-Picture |

```
┌─────────────────────┐         localStorage          ┌─────────────────────┐
│  video_screen.html  │◄──── rugby_time, rugby_cmd ──►│  control_panel.html │
│  (монитор 2)        │                               │  (монитор 1)        │
└─────────────────────┘                               └─────────────────────┘
```

Workflow прототипа: **ВЕХА** → пауза → игрок → действие → outcome → CSV.

Что переносим в продукт: двухэкранный паттерн, кнопочный ввод, таймлайн, горячие клавиши.  
Что меняем: localStorage → нормальная синхронизация окон; данные → БД; игроки/действия → из справочников и шаблона.

## Дополнительные решения

| Тема | Решение |
|------|---------|
| **Команды** | Единый справочник; выбор перед разметкой матча |
| **Сезон** | **Не отдельная сущность** — период времени (фильтр по датам матчей) |
| **Комментарии** | Кнопки-подсказки + ручной ввод для особых случаев |
| **LineupRole** | Флаг `starter` / `substitute` — только UX разметки (группировка кнопок); фильтрация отчётов — не в MVP |
