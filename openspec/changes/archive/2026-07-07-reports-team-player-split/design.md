## Context

Страница `/reports` поддерживает командный и индивидуальный отчёты, drill-down «По игрокам» (inline) и «Комментарии». Lazyweb-мокап `mockup-team-player-split.png` задаёт split-layout для командного отчёта.

**Уточнение продукта (после ревью):** правая панель — только расшифровка действия по игрокам. Полный индивидуальный отчёт — через существующий режим «Индивидуальный» в форме, не через клик по игроку в панели.

## Goals / Non-Goals

**Goals:**

- Командный отчёт: split ~65/35 по мокапу
- «По игрокам» → таблица игроков для действия в правой панели
- Подсветка активной строки действия слева
- Индивидуальный отчёт — без изменений (полная ширина, без панели)

**Non-Goals:**

- Полный отчёт игрока в правой панели
- Удаление «Тип отчёта»
- Другие варианты Lazyweb (матрица, dual-list матчей, sticky)
- Изменение SQL-агрегаций

## Mockup spec

| Элемент | Спецификация |
|--------|----------------|
| Split (только team) | Левая: «Отчёт команды»; правая: «Панель игрока» |
| Плейсхолдер | `Выберите действие` |
| Левая колонка | Матчи, категории, таблицы; кнопки `По игрокам`, `Комментарии` |
| Правая колонка | Заголовок действия + таблица игроков (read-only, без ссылок на full report) |

| Тип отчёта | **Остаётся** (отклонение от мокапа — осознанное) |
| Individual mode | Одна колонка, текущий UX |

## Decisions

### Decision 1: Split только для командного отчёта

`report_type=team` + `generated` → `.report-split`. `report_type=individual` → один `.card` с результатами, как сейчас.

### Decision 2: Панель = action breakdown only

`GET /reports/player-panel?mode=action-breakdown&actionId=…` → `_report_player_detail.html` + заголовок действия/категории/матча.

Строки игроков — обычный текст, **без** HTMX на полный отчёт.

### Decision 3: «По игрокам» → панель, не inline

Убрать inline `<div>` под строкой. Повторный клик по той же строке или кнопка «Скрыть» сбрасывает панель в placeholder.

### Decision 4: Индивидуальный отчёт — отдельный режим

Пользователь переключает «Тип отчёта» → «Индивидуальный», выбирает игрока, «Сформировать». Логика `reports_page` и `report_results` для `individual` не меняется.

### Decision 5: Endpoint `/reports/player-panel`

| mode | Содержимое |
|------|------------|
| `empty` | Плейсхолдер «Выберите действие» |
| `action-breakdown` | Таблица игроков для действия |

`/reports/player-detail` — redirect на panel или удалить после миграции JS.

### Decision 6: CSS в `app.css`

`.report-split`, `.report-split__team`, `.report-split__player`, `.report-player-panel__placeholder`, `.report-action-row--active`.

## Risks / Trade-offs

- **Узкая правая колонка** → `overflow-x: auto` внутри панели
- **Sticky правая панель** на desktop при длинном левом скролле
- **Мокап без «Тип отчёта»** → в UI остаётся переключатель; split виден только в team mode
