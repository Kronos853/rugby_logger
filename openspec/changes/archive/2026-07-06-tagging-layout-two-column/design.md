## Context

Пульт разметки (`templates/tagging/control.html`) — единственная страница без `base.html`/`app-shell`. Текущая структура: узкий sidebar слева (22% ширины) + вертикальная main-колонка (78%), таймлайн всегда внизу потока — за пределами экрана.

Страница не использует backend-изменений: все данные приходят через стандартный GET/POST Flask. JavaScript (`tagging.js`) обрабатывает только BroadcastChannel и seek — логика capture/update не меняется.

Активный стек: Flask + Jinja + HTMX, Alpine.js доступен через CDN в `base.html` (на `control.html` пока не подключён).

## Goals / Non-Goals

**Goals:**
- Таймлайн и зона ввода видны одновременно на экране без прокрутки body
- CSV-операции и навигация убраны из основного потока → toolbar-меню
- Комментарии не занимают центральное место в зоне ввода
- Страница работает на 1366×768 (пульт на ноутбуке)
- Нет изменений в backend-логике, route, JS

**Non-Goals:**
- Изменение логики capture / update / delete событий
- Группировка действий по категориям (tabs/accordion)
- Горячие клавиши
- Мобильный (<900px) responsive

## Decisions

### 1. Страница как `100vh` без прокрутки body

**Решение:** `body { overflow: hidden }` и `height: 100vh` на `.tagging-page`. Toolbar фиксированной высоты (~56px), workspace под ним — `flex: 1; min-height: 0`.

**Почему:** Единственный способ гарантировать, что таймлайн всегда в видимой области независимо от числа игроков, действий и пресетов. Без `min-height: 0` на flex-children браузер не позволяет дочернему элементу иметь `overflow-y: auto` — это фундаментальное ограничение CSS flexbox.

**Альтернатива:** `position: sticky` у таймлайна — не работает в колоночном потоке; `position: fixed` — ломает поведение при частичном обновлении HTMX.

### 2. Две колонки в `.tagging-workspace`

**Решение:** `display: flex; flex-direction: row`. Левая `.tagging-input-col` — `flex: 0 0 45%; overflow-y: auto`. Правая `.tagging-timeline-col` — `flex: 1; display: flex; flex-direction: column; min-height: 0`.

**Левая колонка порядок блоков (сверху→вниз, по частоте):** Игроки → Действия → Outcome → Комментарий.

**Breakpoint `@media (max-width: 1199px)`:** колонки стекаются (`flex-direction: column`), таймлайн получает фиксированную высоту (`max-height: 40vh`), левая — оставшееся пространство.

### 3. Горизонтальный scroll у таймлайна

**Решение:** `.table-wrapper { overflow: auto }` (оба направления). Таблица — `min-width: 900px; white-space: nowrap`. Sticky header работает при обоих типах scroll.

**Приоритет видимости:** Тайм, Время, Игрок/Команда, Действие, Результат — всегда в левой части; Категория, Комментарий — могут уходить вправо.

**Длинные тексты:** `text-overflow: ellipsis` + `max-width` у `td` для Комментария; полный текст по `title` атрибуту.

### 4. Toolbar: единая полоса поверх workspace

**Структура:**
```
[Счёт] | [Матч] | [2т] | [video-time] | [НОВОЕ СОБЫТИЕ] | [⋮ Меню]
```

`⋮ Меню` — нативный `<details>/<summary>` dropdown: Открыть видео, Экспорт CSV, Импорт CSV (form с file input), ← Подготовка. Flash-сообщения после импорта — в toolbar под строкой, не в отдельной card.

**Alpine CDN** не подключать на этой странице; `<details>` не требует JS.

### 5. Компактные комментарии

**Решение:** пресеты — горизонтальный flex с `overflow-x: auto; white-space: nowrap` (одна строка, scroll при >5–6 кнопок). Ниже — поле ввода с сохранением on-submit (кнопка «Сохранить» остаётся — POST форма без HTMX, как сейчас).

**Почему:** `flex-wrap: wrap` при 10 пресетах даёт ~3–4 строки = ~100px дополнительной высоты. Горизонтальный scroll одной строкой — ~36px.

### 6. `form-scroll.js` якорь

**Текущее поведение:** после POST scroll восстанавливается по `scrollY` — работает при вертикальной прокрутке всей страницы. После перехода на `overflow: hidden` body прокручиваемыми станут только `.tagging-input-col` и `.table-wrapper`.

**Решение:** обновить `form-scroll.js` — сохранять/восстанавливать `scrollTop` у `.tagging-input-col` вместо `window.scrollY`.

## Risks / Trade-offs

| Риск | Митигация |
|------|-----------|
| На 1366px левая колонка (~620px) может не вместить 30 кнопок действий без scroll | Внутренний scroll левой колонки уже предусмотрен (overflow-y: auto) |
| Sticky header таблицы при горизонтальном scroll может вести себя по-разному в браузерах | Тест в Chrome/Edge; fallback — убрать sticky, принять нефиксированный заголовок |
| `<details>` dropdown не закрывается при клике вне — ожидаемое поведение нативного элемента | Принято; альтернатива JS click-outside — за рамками scope |
| HTMX partial refresh (timeline) заменяет `#timeline-panel` — контейнер должен оставаться в правой колонке | `_timeline.html` и `hx-target` не меняются; структура правой колонки это обеспечивает |
| `form-scroll.js` изменение ломает scroll на других страницах | Изменение сделать только для `.tagging-input-col`; на других страницах класс отсутствует → поведение не изменится |
