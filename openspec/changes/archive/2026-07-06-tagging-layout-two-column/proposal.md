## Why

Пульт разметки — вертикальная колонка: таймлайн всегда ниже fold, пользователь вынужден прокручивать страницу после каждого события, чтобы проверить результат. На ноутбуке или малом экране (1366×768) таймлайн не виден совсем. Это критично при быстрой разметке в реальном времени.

## What Changes

- Страница переходит на двухколоночный layout, занимающий `100vh` без прокрутки body
- Левая колонка — зона ввода (игроки, действия, outcome, комментарии), с вертикальным scroll внутри
- Правая колонка — таймлайн, занимает всю высоту, с вертикальным и горизонтальным scroll при нехватке ширины
- Toolbar поверх обеих колонок: счёт, матч, тайм, время видео, кнопка «Новое событие», меню «⋮» (видео, CSV экспорт/импорт, ← Подготовка)
- Старый боковой sidebar (сайдбар) удаляется
- Комментарии — компактный блок в нижней части левой колонки (до ~10 пресетов в одну строку с горизонтальным scroll или dropdown)
- Breakpoint для узких экранов (<1200px): колонки стекаются вертикально (таймлайн вверху, ввод ниже)

## Capabilities

### New Capabilities

_Нет новых capabilities: изменение затрагивает только layout и UX существующей функциональности._

### Modified Capabilities

- `video-tagging`: требование о видимости таймлайна без прокрутки страницы во время разметки

## Impact

- `templates/tagging/control.html` — полный переворот структуры
- `static/tagging.css` — новые классы layout, toolbar, двух колонок, компактных комментариев, breakpoint
- `static/form-scroll.js` — якорь прокрутки изменить с body на `.tagging-input-col`
- `templates/tagging/_timeline.html` — без изменений в логике, только контейнер
- Без изменений backend, routes, JS логики capture/update
